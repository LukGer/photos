# Animation plans

Generated by `improve-animations` against commit `4d82436`. Each plan is
self-contained — an executor with no prior context can implement it. Plans only
touch motion properties; none change app logic or data.

The gallery's motion is already well-crafted (strong easing tokens, proper
`useReducedMotion`, gated hover, subtle press feedback, spring-based tilt). These
three plans are refinements, not rescues.

## Recommended execution order

Any order works — the three plans touch disjoint files with **no dependencies**
between them. Suggested order by user-visible impact:

1. **001** — biggest perceived win; removes the jarring photo pop on the detail page.
2. **002** — prevents late gallery cards from being invisible for seconds; fixes a
   reduced-motion leak.
3. **003** — one-line performance/correctness cleanup.

Each can be committed independently.
